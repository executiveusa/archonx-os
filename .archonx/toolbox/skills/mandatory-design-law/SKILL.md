# Mandatory Design Law Skill

## Authority
This skill is mandatory for any frontend, visual, product UI, landing page, dashboard,
brand surface, or interaction design work inside ArchonX and any managed repo.

This is not optional guidance. It is policy.

## Core Law
Every interface must satisfy two standards at the same time:

1. **Intentional visual design**
2. **Steve Krug usability logic**

If a design is visually flashy but unclear, it fails.
If a design is clear but generic, lifeless, or unbranded, it fails.

## Steve Krug Logic As Law
Use these as hard constraints:

- Do not make the user think unnecessarily.
- Each screen must communicate purpose quickly.
- Navigation must feel obvious, not clever.
- Important actions must look important.
- Clickable things must look clickable.
- Users must be able to scan instead of read everything.
- Reduce noise, reduce choices, and remove avoidable friction.
- Prefer convention when convention improves clarity.
- Every page should answer:
  - Where am I?
  - What is this?
  - What can I do here?
  - Why should I care?

## ArchonX Visual Standards
- No generic AI-slop layouts.
- No lazy default stacks or interchangeable startup UI.
- Typography must feel chosen on purpose.
- Visual hierarchy must be obvious in under five seconds.
- Motion must support comprehension, not decoration.
- Color must support meaning, contrast, and brand identity.
- Backgrounds should feel composed, not flat filler.
- Mobile and desktop both matter; neither can be an afterthought.
- Empty states, error states, and loading states must be designed intentionally.

## Anti-Patterns
Reject work that includes:

- confusing navigation
- decorative clutter without purpose
- low-contrast text
- weak hierarchy
- hidden primary actions
- over-animated gimmicks
- generic card soup
- meaningless gradients with no structure
- default-feeling typography and spacing
- pages that require explanation to understand

## Required Design Review Questions
Before shipping any frontend or visual work, answer:

1. What is the page trying to accomplish?
2. What should the user notice first?
3. What is the single clearest next action?
4. Can a first-time user understand this page in seconds?
5. Does the visual style feel deliberate and branded?
6. Is the mobile layout equally clear?
7. Is there anything decorative that can be removed without loss?

## Delivery Standard
Frontend and visual work must produce:

- a clear information hierarchy
- a visible primary action
- responsive behavior
- accessibility-conscious contrast and structure
- intentional typography, spacing, and composition
- a short design rationale explaining why the chosen direction is correct

## Enforcement
- Any agent doing frontend or visual work must load this skill first.
- AGENTS doctrine references this skill as mandatory.
- Reviewers should reject UI work that violates this skill even if the code functions.

## Short Rule
Make it clear. Make it intentional. Make it easy to use. Do not make the user think.
